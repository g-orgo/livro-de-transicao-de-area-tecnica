
# Capítulo 0 — Por que Python?

## Objetivos

Construir um modelo mental correto: Python não domina IA por ser rápido em loops puros, mas por ser uma camada expressiva de integração sobre bibliotecas nativas, GPUs e serviços.

## Mentalidade Python

Python privilegia produtividade, legibilidade e interoperabilidade. A pergunta não é “Python faz a multiplicação de matrizes mais rápido?”, mas “qual camada deve orquestrar o cálculo e qual deve executá-lo?”.

## Comparando com JavaScript

Node também integra bibliotecas nativas, mas em IA o ecossistema Python historicamente concentra interfaces para NumPy, PyTorch, CUDA e ferramentas científicas.

## Conceitos

- Interpretador, extensões nativas e vetorizar operações.
- Ecossistema como multiplicador de produtividade.
- Zen of Python: legibilidade e convenções.

## 🧠 Como um Pythonista resolveria isso

Em vez de escrever um loop Python para processar milhões de números, delega a operação em lote a uma biblioteca especializada como NumPy ou PyTorch.

## Implementação

Comparar conceitualmente um loop Python com uma operação vetorizada. Nenhum código do PyTool é criado neste capítulo.

## Engenharia

Separar orquestração de trabalho intensivo é uma decisão de arquitetura, não uma limitação a esconder.

## ⚠️ Erros comuns

- Concluir que Python é “rápido em matemática” sem identificar qual código realmente executa a operação.
- Escolher uma linguagem apenas por benchmark isolado.

## Exercícios

1. Por que um ecossistema grande pode ser mais valioso que desempenho bruto para um produto?
2. Onde você espera que Python orquestre trabalho, e onde espera código nativo?
